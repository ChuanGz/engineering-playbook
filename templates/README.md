# Templates

Use these purpose-specific formats to make consequential engineering work reviewable. Remove inapplicable sections instead of adding placeholders; never invent evidence to complete a template.

| Template | Decision supported |
| --- | --- |
| [Decision record](decision-record.md) | Preserve a consequential choice, alternatives, and consequences. |
| [Design review](design-review.md) | Decide whether a design is ready for implementation. |
| [Requirement review](requirement-review.md) | Decide whether a requirement is ready to estimate and design. |
| [Risk assessment](risk-assessment.md) | Accept, reduce, transfer, or avoid a named risk. |
| [Estimation](estimation.md) | Communicate scope, range, assumptions, and confidence. |
| [Test strategy](test-strategy.md) | Map material failure risks to credible evidence. |
| [Code review](code-review.md) | Record the risk-based disposition of a change. |
| [Incident review](incident-review.md) | Learn from system conditions without assigning personal blame. |
| [Retrospective](retrospective.md) | Select one evidence-backed improvement to team delivery. |
