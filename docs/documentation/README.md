# Documentation

## Purpose

Preserve useful engineering context.

## Core guidance

Write for a named audience, keep decisions discoverable, validate links, assign ownership, and remove stale guidance.

Documentation is maintained engineering infrastructure. A document is justified by the task, decision, contract, or recovery action it enables—not by completeness of a folder tree.

## Guides

- [Documentation principles](documentation-principles.md) — create the smallest artifact that preserves necessary knowledge.
- [README standard](readme-standard.md) — make a repository understandable and actionable from its landing page.
- [Decision records](decision-records.md) — preserve consequential choices without rewriting history.
- [Diagrams](diagrams.md) — use visuals only when relationships are harder to understand in prose.
- [Documentation review](documentation-review.md) — verify correctness, usefulness, ownership, and maintenance cost.

## Review questions

- Can the named reader complete a task or decision using this document?
- Do material claims trace to authoritative evidence or labeled assumptions?
- Does the document add distinct value instead of copying another source of truth?
- Which owner or change event will update, supersede, or remove it?
