# Requirement Analysis

## Purpose

Clarify outcomes before implementation.

## Core guidance

Confirm stakeholders, constraints, acceptance criteria, dependencies, gaps, risks, estimates, traceability, and change impact.

## Guides

- [Requirement review checklist](requirement-review-checklist.md) — determine whether a requirement is ready for a delivery decision.
- [Ambiguity detection](ambiguity-detection.md) — expose language that permits conflicting interpretations.
- [Scope breakdown](scope-breakdown.md) — split outcomes into independently reviewable delivery slices.
- [Estimation strategy](estimation-strategy.md) — estimate from explicit scope, assumptions, and uncertainty.
- [Validation before implementation](validation-before-implementation.md) — obtain evidence before committing implementation effort.
- [Common failures](common-failures.md) — recognize recurring requirement-analysis breakdowns.

## Review questions

- Is the context and intended outcome explicit?
- Are alternatives and trade-offs recorded?
- Can the decision be validated and maintained?
